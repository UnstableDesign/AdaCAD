New Features

  • Added Library Design Mode: A place to manage all drafts and view as grid (helpful when assigning drafts to images or importing/exporting drafts for use with Photoshop).
  • Import .WIF or Bitmap files: You can upload a single or many individual .wif files or bitmaps.
  • Simulation: new approach developed based on TopoKnit algorithm (which made it much faster) and custom modifications to detect layer relationships. still in progress.
  • Coloring Pages: A silly/fun feature that allows you to print a coloring page of any given draft.
  • Speed and Performance Improvements: We did a major redesign of how drafts are computed and rendered to prevent crashing and slowdowns! New approach to undo makes it faster and more reliable.
  • Better Visibility and Feedback on Mixer: Added more bolding on connections going into and out of selected operations. We also made zoom variable, so that text stays more readable when zoomed out. Prevented drafts from moving outside selectable range and, if it happens again, you can use “Bump” command-b to move the whole workflow down and to the right until it is back in view. Any draft that is being recomputed flashes to black when changes are made.
  • Mouse Pan and Zoom: Enabled by community members! This lets you zoom in and out of the workspace using a mouse-wheel or pinch and expand gestures on a tablet or computer. Pan via ctrl + click. Thank you Patrick Steppan!
  • New Operations: Integrated 2-3 years of community created operations as well as a few more.
    • Bitfield: a mathematical function that uses x/y values to return a boolean value for each cell, to make 'bitfield' patterns
    • Random Tree: a random tree structure
    • Sierpinski Square: a Sierpinski square structure
    • Create Material Sequence: a sequence of materials specified by a list of numbers
    • Threading/Treadling Sequence: a sequence of threading/treadling specified by a list of numbers
    • Tile: added the ability to offset rows/columns when tiling
  • Text Fields can Accept .CSV input: so if you want to make a long list of inputs from a database (e.g. climate data, etc) this is easier!
  • Draft Editor: reworked UI to make select/copy/paste actions more accessible
